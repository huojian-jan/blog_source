我的工作岗位是 Windows 开发工程师，常用的技术栈是 WPF、C#、.NET和其他的 Windows 相关的技术。所以我的主力机是 Windows，甚至我自己组装了一台高性能的 PC 作为我的主力机器，公司发的笔记本只是在开会的时候使用。后面公司给每个人配了 Cursor账号，后面我为了全面拥抱 AI,抛弃了我最熟悉的 Visual Studio+Resharper 的工作方式，换到了 Cursor 上，在不需要写 UI 的时候我很少回打开 Visual Studio,甚至调试的工作也都放在 Cursor 上完成了。后来，公司调整了组织架构，我被分配在业务组，写UI 的机会也很少了，所以我后面基本上就不打开Visual Studio 了。因为 Cursor 是基于 vscode 开发的，换到 Cursor 之后，之前的很多需求，其实都可以通过几个插件的组合来解决，快捷键也重新调教了一番，算是比较顺手了。
回老家过年的时候，我带上了我平时很少用的 M1 Macbook Air,然后这段用 Macbook 的这段时间，我突然萌生了把主力机换成 Macbook 的念头。我在火车上，等飞机的一些碎片时间搜索了相关的技术方案，怎么做到主力机器换成 Macbook 之后，如何完成我的本职工作，会不会影响我的工作效率。回来开工之后，我就带着我的 Macbook Air 去上班了。

我尝试过的几个方案：
1. Syngery。这个方案是我在火车上搜索到的，Syngery 是一个跨平台的同步工具，可以同步 Windows 和 Macbook 的文件，而且支持文件的实时同步，但这个是商业软件，需要购买 License,而且还不便宜。
2. Barrier。Barrier 是 Syngery 的开源实现版本，但是我用下来不稳定，尤其是在不同的屏幕之间滑动鼠标的时候没有处理好，体验很差，而且容易断联，需要重新从宿主机，移动鼠标到目标主机。


3.用 MouseShare,但是 MouseShare 的稳定性不是很好，而且同步剪切板需要购买企业版。我大概用了 2 天时间，放弃了，因为同步剪切版是刚需，不能接受。

4. ParSec。ParSec是为了同步游戏设计的软件，可能是 P2P 的网络设计，延迟控制的缺失很好，也可以同步剪切板，但是有个问题我不能接受，那就是我在 Windows 机上滚动代码的时候会出现代码从模糊到清晰的一个过程，经常会降低分辨率，看着严谨很累，尤其是我从 Macbook 的高品质屏幕抬头看到 Windows 远程的屏幕的时候，对比很明显。虽然功能需求都满足，但是现实的效果很难接受。

5.UU远程。UU 远程是网易开发的远程工具，最高支持 4K 的分辨率，延迟极低，基本上解决了我前面提到的几个工具的所有缺点，我用了差不多一周的时间。直到后面我遇到了 Macbook 上支持 RDP 协议的远程工具，我才放弃了 UU 远程。

6. Macbook 上支持 RDP 协议的远程工具。我尝试了好几款，最后选择了 Microsoft Remote Desktop,这款软件是微软官方开发的，因为是局域网直连的，所以几乎没有延迟，而且 RDP 协议的加持下，画面的渲染跟真机几乎没有区别。

我之前的工作机器是 Windows11 的台式机，配了两个屏幕，一个横屏和一个竖屏。后来我把竖屏的屏幕退了，横屏显示器接到了我的 Macbook 上，我通过 RDP 协议连接到我的 Windows 机器上，外接显示器上显示远程的 Windows 桌面，这样上下的双屏非常人体工学没效率极高，而且撤掉了一个显示器，桌面变得非常整洁，很宽敞。再后来，我把之前公司发的 Windows 笔记本退了，刚好有新采购的 M5 Pro 的新机器，然后我就把自己的 Macbook Air 拿回家了，用公司配的 M5 Pro 远程我的 Windows 机器，这样我就有了一台 Macbook 和一台 Windows 机器，而且都是最高配置的机器，性能非常强悍。

最近在做 AI 项目，在测试不依赖 Windows 环境的情况下，我现在都习惯使用 Macbook 来完成了，Macbook 类 Unix 的设计，让我在开发过程中非常舒适，尤其是 Cursor 的加持下，我基本上可以完成我的所有工作，而且效率非常高。
